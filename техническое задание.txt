    Техническое задание на разработку бизнес-модели с двумя сущностями.

    В рамках задания были разработаны две сущности "Должность" и "Сотрудник" с отношением "один ко многим".
1. Сущность "post" - описывает данные о должностях
    idPost: целочисленный уникальный идентификатор должности
    name: название должности (строка)
    salary: размер заработной платы на этой должности (число с плавающей запятой)
    employees: список сотрудников, занимающих данную должность (коллекция объектов "Сотрудник")
2. Сущность "employee" - хранит данные о сотрудниках
    idEmployee: целочисленный уникальный идентификатор сотрудника
    fio: ФИО сотрудника (строка)
    dateOfBirth: дата рождения сотрудника (дата)
    education: образование сотрудника (строка)
    experience: опыт работы сотрудника (целое число)
    post: должность, занимаемая сотрудником (ссылка на объект "Должность")

    Требования к системе
1. Возможность добавления, редактирования и удаления данных о должностях.
2. Возможность добавления, редактирования и удаления данных о сотрудниках, включая назначение на должность.

Пример JSON-представления данных
[
    {
        "idEmployee": 3,
        "dateOfBirth": "1990-07-20T14:56:55.000+00:00",
        "education": "Высшее",
        "experience": 5,
        "post": {
            "idPost": 2,
            "name": "Разработчик",
            "salary": 4000.0
        },
        "fio": "Иванов Иван Иванович"
    }
]

    Предполагается использование RESTful API для взаимодействия с системой.
    Доступные HTTP-методы: GET, POST, PUT, DELETE.
    Формат данных: JSON.

    Технологии и инструменты
Платформа: Spring Boot
База данных: PostgreSQL
Язык программирования: Java
Инструменты разработки: IntelliJ IDEA, Git, GitHub, Maven, hibernate.

    Этапы разработки
1.Проектирование и моделирование сущностей "Должность" и "Сотрудник".
2.Реализация REST-контроллеров для управления данными.
3.Интеграция с базой данных PostgreSQL.
4.Разработка бизнес-логики для взаимодействия с сущностями.
5.Реализация API-методов для выполнения CRUD-операций.
6.Тестирование и отладка системы.
7.Развертывание приложения и документирование API.
